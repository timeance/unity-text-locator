# 项目级 STYLE_GUIDE 模板

并行翻译时，把本模板复制为 `<PROJ>/skill/references/STYLE_GUIDE.md` 并填入本书信息（`<…>` 处）。
目的：让所有并行 subagent 产出**完全一致**的风格——锁定表管「词」，本指南管「排版与判断」。
（串行翻译也可用它来固定风格。）下列示例名（Marlow、Vance…）为通用占位，与任何具体作品无关。

# 翻译风格指南（《<书名>》→ <目标语言> 译本）

前 <N> 章已由主控翻译并经用户确认。**你的译文必须与既有风格完全一致。**
锁定词汇表（`<book>.glossary.locked.json`）是人名/术语的唯一权威来源，本指南为其补充与兜底。
基本原则同 AiNiee 标准提示词：逐行翻译、保留标记/占位符、忠实准确。**人名、头衔、风格等领域规则以本项目的用户自定义提示词（`user_prompt.md`）为准**——下面把要点摘录于此，供并行 agent 对齐。

## 0. 最重要：逐行对应
输入是 `{text_index, source_text}` 数组；输出等长 `{text_index, translated_text}` 数组，index 原样、顺序不变。
**绝不合并/拆分段落，绝不漏译任何从句/定语/状语。** 每段译完对照源文自检。

## 1. 人名 / 专有名词
- **人名按锁定表 `render`**（默认保留原文原样，具体以本项目用户提示词为准）。撇号用源文形式（如 `Da'ren` 的右单引号 ’）。
- **专有名词**（地名、组织名等）：默认保留原文；锁定表给了中文译名（或对应真实存在、有通用译名的事物）则用中文。
- 把保留原文的源文/术语嵌入目标语言文本时，按目标语言排版惯例处理（如中文：两侧各加一个半角空格，`Marlow 上校`）。

## 2. 头衔 / 称谓的处理（按本项目需求填写；无则删）
头衔规则**不由本技能预设**——从本项目的用户提示词（`user_prompt.md`）摘录要点，供 agent 对齐。例如：
- 是否后置、各军衔/官职/尊称的固定译法（若本书无军衔/头衔体系，整节删去）。
- 同一称谓在不同语境的取舍（例：`Captain` 指挥本舰→舰长，其他语境→上校）。

## 3. 族群 / 阵营 / 地名
- **有先例/常见的主要专名用其译名**（沿用锁定表与既往译本，全书统一）。
- **新引入、无先例的专名默认保留原文**（不自创音译）——除非用户另有指示。
- 组织名多为「描述性意译」；地名有通用译名则用，否则保留原文。

## 4. 术语一致
一律查锁定表 `terms`；`keep_source:true` 的保持原文。计量/等级等按锁定表统一。

## 5. 标点 / 排版（以下为**中文目标语言**惯例；其他目标语言按其规范替换）
- 对话用全角弯引号 “ ”。
- 强调 / 绰号 / 内层引用用 『 』。
- 书名 / 篇目名用 《 》（**标题文本本身保留原文**）。
- 日期行：`Month D, YYYY` → `YYYY 年 M 月 D 日`。
- 章节号、地点/标识标题行：保留原样英文与编号。
- 数字、占位符、`<i>` 等标签原样保留。

## 6. 文风
流畅、典雅的目标语言文学叙事；保留原文语气。长句可按目标语言习惯切分，但不得漏意。对照已确认章节的语感。

## 7. 新实体处理
锁定表里没有的专有名词 → **默认保留原文原样**，并记录到你的 `new_terms` 输出文件（每行一个），供主控统一并入词汇表。

## 8.（同名词陷阱）
注意一词多义：作品里的**专有名词**有时与作品名/系列名/品牌名同形。作品名/系列名的引用应保留原文（放进《》），
只有指作品内实体时才套用专名规则。OCR 粘连会把专名和邻词连在一起，按译文（而非源文）做这类判断更可靠。

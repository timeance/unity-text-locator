# 翻译规则（AiNiee 原生提示词 + 管线适配）

agent 翻译时遵循的规则**就是 AiNiee 自带的标准翻译提示词**，外加本管线（`batch read`/`write`）必须满足的逐行索引对齐。
具体的人名、术语译法不写死在规则里，而是以项目锁定表（**术语表 / 角色表**）为唯一真相源——这与 AiNiee 的做法一致。

**语言无关**：支持任意 `{source_language}` → `{target_language}`；下述原则不绑定特定语言，目标语言相关的标点/排版/本地化习惯写在用户的风格指南/提示词里。

---

## AiNiee 原生翻译原则

你是一名专业的翻译家，任务是把 `{source_language}` 文本翻译成 `{target_language}`：

- **逐行翻译，不要合并**；原样保留文本中的序号、标记符、占位符、换行符、转义符、代码调用过程等特殊内容，保持原来的格式。
- **忠实准确**地进行翻译。原作是伟大的艺术作品，允许使用直白或粗俗的描述，不应随意删减或篡改，不应回避或淡化，以保持作品的原貌。

> 可选「深思（CoT）」流程：① 先逐行直译；② 再结合语义语境、专业术语、上下文、人物设定等校正；③ 最后整合为自然流畅、符合目标语言表达习惯的意译润色稿。

---

## 本管线的硬性要求

- **逐行索引对齐**：输入是 `batch read` 输出的 `{text_index, source_text}` 数组；写回时输出等长的 `{text_index, translated_text}` 数组，`text_index` 一一对应、顺序不变。**绝不合并 / 拆分段落，绝不漏译任何从句**（否则写回缓存会错位）。
- **保留标记**：`<i></i>`、`<b></b>` 等富文本标签、占位符（`{0}`、`%s`）、转义序列（`\n`）一律原样保留、位置不变（epub 渲染依赖它）。
- **术语 / 名称一致**：一切以锁定表为准——`terms` 用 `dst`，`keep_source:true` 的保持原文；`characters` 出现时用其 `render` 值；同一名称全书统一。锁定表外的新专名默认保留原文，并记入待审。
- 源文偶有 OCR 粘连词（多个词无空格连在一起，如 `centralavenue`）：按语义还原词边界后翻译，并在 `needs_review` 标记该段。

---

## 写回前自查清单

- [ ] 输出数组长度 == 输入数组长度，`text_index` 一一对应、顺序不变
- [ ] 所有标签 / 占位符 / 转义序列原样保留
- [ ] 术语用锁定表 `dst`（`keep_source:true` 保持原文）；人名用锁定表 `render`
- [ ] 无漏译子句（对照源文检查从句 / 定语）
